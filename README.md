# Kernel-sandboxed agent + privileged broker (UDS)

> **Work in progress â€” proof of concept.** This project is experimental and not production-ready. APIs, sandbox policies, and broker behavior may change without notice. Use it to explore the architecture and provide feedback; do not rely on it for security-critical workloads yet.

A **sandboxed agent process** runs with a scrubbed environment and kernel filesystem restrictions. Sensitive tools (`git`, `mvn`, `curl` with placeholder keys, broker skills) go through **interceptors** to a **privileged broker** over a Unix domain socket. The broker injects secrets, runs the real command, and scrubs output before the LLM sees it.

## Quick start

```bash
./bin/setup
# Edit ~/.config/agent-broker/env â€” set BROKER_ALLOWED_ROOTS (see table below)
./bin/doctor
./bin/agent pi            # or: ./bin/agent bash | ./bin/agent opencode
```

After `git pull`: `./bin/setup --check`

**Smoke tests (no LLM):** `./bin/doctor`, `./bin/agent-run pi --help`, `./bin/agent-run pi list`, or `./bin/agent-run bash -lc 'entur-departures jernbanetorget 1'`. Avoid `pi -p` â€” it blocks on the API.

The broker starts **on demand** (`./bin/agent`, `./bin/doctor`). No separate terminal needed.

## Configuration

All settings live in `~/.config/agent-broker/env` (created by `./bin/setup`).

| Variable | Required | Purpose |
|----------|----------|---------|
| `BROKER_ALLOWED_ROOTS` | **yes** | Comma-separated dirs the broker accepts as `workdir` â€” include every project where you run `./bin/agent` |
| `BROKER_SOCKET_TOKEN` | auto | Generated by `setup`; required on broker requests when set |
| `PROD_API_KEY` | for curl | Replaces `X-PROD-KEY` in intercepted `curl` requests |
| `BROKER_SOCK` | no | Override socket path (defaults below) |
| `BROKER_AUDIT_LOG` | no | JSONL audit trail (default `~/.config/agent-broker/audit.jsonl`) |
| `BROKER_SCRUB_PII` | no | Set `0` to disable regex PII scrubbing on broker output (default on) |
| `SUBRO_SANDBOX` | no | `native` (default) or `cplt` â€” see [docs/cplt.md](docs/cplt.md) |
| `SUBRO_CPLT_BIN` | no | Absolute path to `cplt` when not on default `PATH` |
| `ENTUR_CLIENT_NAME` | no | Client name for the Entur showcase skill (default `subro-agent-broker`) |

Debug without kernel sandbox: `SUBRO_NO_SANDBOX=1 ./bin/agent bash` (env scrub + broker still apply).

## How it works

Designed for running CLI coding agents safely (prompt injection, malicious repos, untrusted build output).

- **Secret isolation** â€” agent runs under `env -i` with a fake `HOME`; secrets live only in the broker env file.
- **Kernel filesystem boundaries** â€” read/write the repo + broker socket; denied `~/.ssh`, `~/.m2`, shell dotfiles, and repo `.env*` / `.npmrc` from the sandboxed agent.
- **Tool mediation** â€” `PATH` prefers interceptors for `git`, `mvn`, `curl`, and broker skills.
- **Output scrubbing** â€” per-tool secret redaction plus a regex PII pack (`broker/scrub.py`, on by default).

**Non-goals:** this does not make arbitrary code execution safe. The broker is the trust boundary â€” keep its allowlist small. Repo `.env` files work for humans but are hidden from the sandboxed agent; put high-privilege secrets in the broker env file.

## Harness integration (pi / OpenCode / Cursor)

Run from any project directory under `BROKER_ALLOWED_ROOTS` â€” not only this repo.

```bash
./bin/skills-sync                    # .cursor/, .pi/, .opencode/, .agents/ symlinks
./bin/skills-sync --global-pi        # merge skills/ â†’ ~/.pi/agent/settings.json
./bin/skills-sync --global-opencode  # symlink skills/ â†’ ~/.config/opencode/skills/
./bin/agent pi
./bin/agent opencode
```

Each `./bin/agent <harness>` mode:

- starts the broker on demand
- runs `skills-sync` when `skills/` changed (stamp: `.broker/.skills-sync-stamp`; override with `SUBRO_FORCE_SKILLS_SYNC=1`)
- symlinks the CLI (+ `node`) into `./.agent-bin/`
- points harness config at your **real** `~` dirs (sandbox uses fake `HOME=./.agent-home`)

| Harness | Gotcha | Fix |
|---------|--------|-----|
| **pi** | fake `HOME` hides `~/.pi/agent` | automatic via `./bin/agent pi` (`PI_CODING_AGENT_DIR`) |
| **pi** | loopback at startup (TUI, `-p`) | Seatbelt `network-bind` on localhost; cplt `allow_localhost_any` |
| **OpenCode** | fake `HOME` hides config/data dirs | automatic via `./bin/agent opencode` (`OPENCODE_*_DIR`) |
| **OpenCode** | global skills | `./bin/skills-sync --global-opencode` |
| **Cursor** | project skills | `./bin/skills-sync` â†’ `.cursor/skills/` |

**Global pi:** run `./bin/skills-sync --global-pi` or add `"skills": ["/path/to/subro/skills"]` to `~/.pi/agent/settings.json`.

**Global OpenCode:** `./bin/agent opencode` sets real home paths for config/data/cache/state. Run `./bin/skills-sync --global-opencode` for global skills.

### Broker lifecycle

Manual control (usually unnecessary â€” broker is on-demand):

```bash
export BROKER_ENV_FILE="$HOME/.config/agent-broker/env"
./bin/broker-daemon start
./bin/broker-daemon status   # detects stale pid files and orphan sockets
./bin/broker-daemon stop
```

Logs: `./.broker/log.txt`. Stale socket? Run `start` again â€” it removes the old socket before binding.

### Sandbox backends

| Backend | Set via | Implementation |
|---------|---------|----------------|
| **native** (default) | `SUBRO_SANDBOX=native` or unset | macOS Seatbelt; Linux `landlock-restrict` when on `PATH` |
| **cplt** (optional) | `SUBRO_SANDBOX=cplt` | [Ceiku/cplt](https://github.com/Ceiku/cplt) â€” see [docs/cplt.md](docs/cplt.md) |

If `cplt` is missing or exits non-zero, subro falls back to native. On Linux without `landlock-restrict`, native runs without kernel sandbox (warning printed). Override: `SUBRO_LANDLOCK_RESTRICT=/path/to/landlock-restrict`.

```bash
./bin/agent-run bash -lc 'which mvn; mvn -v'
./bin/agent-run bash -lc 'curl -sS -H "Authorization: Bearer X-PROD-KEY" https://example.com | head -c 200; echo'
```

- `git` / `mvn` â€” intercepted; broker runs with host `HOME` / `~/.m2`.
- `curl` â€” intercepted only when args contain `X-PROD-KEY`; otherwise runs in sandbox.

### Socket path defaults

- **macOS**: `~/Library/Application Support/agent-broker/agent-broker.sock`
- **Linux**: `$XDG_RUNTIME_DIR/agent-broker/agent-broker.sock` (fallback `/tmp/agent-broker/agent-broker.sock`)

### Adding a new skill

```bash
./bin/skill-init my-api-skill
# edit broker/skills/my_api_skill.py (TOOL_NAME + run())
./bin/broker-daemon stop && ./bin/broker-daemon start
./bin/doctor
```

Skills in `broker/skills/*.py` auto-register if they export `TOOL_NAME` and `run(args)`. Add `interceptors/<name>` (from `skill-init`) and `skills/<name>/SKILL.md`.

### Security notes

- Fake `HOME` at `./.agent-home` â€” host dotfiles not visible to the agent.
- Broker validates `workdir` via `realpath` against `BROKER_ALLOWED_ROOTS`.
- Broker uses `subprocess.run([...], shell=False)` only.
- Output scrubbed for tool secrets and (by default) common PII patterns.
