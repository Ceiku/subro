# P1.3 + P4 implementation guide

## P1.3 — Harness skill loading

After adding or changing skills under `skills/`:

```bash
./bin/skills-sync
```

This:

- Symlinks each `skills/<name>/` → `.cursor/skills/`, `.pi/skills/`, `.opencode/skills/`, `.agents/skills/`
- Creates `.pi/settings.json` with `"skills": ["./skills"]` (pi)
- `--global-pi` merges `skills/` into `~/.pi/agent/settings.json`
- `--global-opencode` symlinks skills into `~/.config/opencode/skills/`
- Writes `.cursor/rules/subro-skills.mdc` (Cursor)
- Regenerates root `AGENTS.md` index

`./bin/agent` runs `skills-sync` automatically before launching your CLI when `skills/` changed
(stamp: `.broker/.skills-sync-stamp`; force with `SUBRO_FORCE_SKILLS_SYNC=1`).

### pi

Run `./bin/agent pi` (not `pi.dev`). Uses [pi skills discovery](https://badlogic-pi-mono.mintlify.app/coding-agent/skills): `.pi/skills/`, `skills/`, and optional global `~/.pi/agent/settings.json`. Sets `PI_CODING_AGENT_DIR` because the sandbox uses a fake `HOME`.

### OpenCode

Run `./bin/agent opencode`. Sets `OPENCODE_CONFIG_DIR`, `OPENCODE_DATA_DIR`, `OPENCODE_CACHE_DIR`, and `OPENCODE_STATE_DIR` to real home paths. Skills: `.opencode/skills/`, `.agents/skills/`, or global `~/.config/opencode/skills/` ([docs](https://opencode.ai/docs/config/)).

### Cursor

Uses `.cursor/skills/<name>/SKILL.md` (project skills).

## P4.1 — Runtime `register-skill`

Register a script the agent wrote under `.agent-skills/`:

```bash
mkdir -p .agent-skills
# agent writes .agent-skills/my-tool.py
register-skill '{"name":"my-tool","script_relpath":".agent-skills/my-tool.py","required_secrets":["MY_API_KEY"]}'
```

Broker validates path, checks secrets exist in **broker env**, persists to `~/.config/agent-broker/registered-skills.json`, writes `.agent-bin/my-tool` shim.

Restart broker after registration (or call `list-tools` via broker — registry reloads on register).

## P4.2 — Agent package manager

```bash
./bin/apm install              # all packages/ with apm.yml
./bin/apm install entur-departures
./bin/apm list
```

Lockfile: `subro.lock.json`

Package manifest: `packages/<name>/apm.yml`

## P4.3 — PII scrubber

Enabled by default (`BROKER_SCRUB_PII=1`). Strips emails, bearer tokens, `password=`, `api_key=`, common token patterns from all broker stdout/stderr.

## P4.4 — STDIO broker transport

Alternative to UDS (e.g. MCP sidecars, subprocess pipes):

```bash
echo '{"tool":"list-tools","args":[],"workdir":"'"$(pwd)"'"}' | ./bin/broker-stdio
```

Same JSON request/response shape as the Unix socket protocol.

## cplt sandbox (default)

Ceiku/cplt is the default backend. Install per [cplt.md](cplt.md). Set `SUBRO_SANDBOX=native` to
use Seatbelt/Landlock instead.
