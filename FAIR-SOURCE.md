Without a separate commercial agreement with the licensor, you **may not** offer
subro (or something with substantially similar functionality) to third parties
as a commercial product or service when it:

1. Substitutes for subro
2. Substitutes for other Ceiku products built on subro that existed at release
3. Offers the same or substantially similar functionality as subro

Examples that typically require a **commercial license**: hosted “agent sandbox +
broker” SaaS for customers, OEM redistribution as your core product, competing
managed service built primarily on this codebase.

## Delayed Open Source (DOSP)

Each version of subro you receive under FSL becomes available under **Apache 2.0**
on the **second anniversary** of the date that version was first made available.

Track release dates via git tags or GitHub releases. After DOSP, Competing Use
restrictions for that version no longer apply.
